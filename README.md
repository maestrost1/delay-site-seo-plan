# delay.site SEO Operating Plan

Статус: утверждено Алексеем 2026-04-14
Цель: вывести delay.site на 1000+ уников в день

## 1. Главные принципы

- SEO для delay.site ведётся системно, а не хаотично.
- `keys.so` полностью ведёт Кими; Алексей ничего вручную не смотрит.
- Перед любым выбором темы обязательно сверяться с `memory/delay-site-content-log.md`.
- Нельзя публиковать тему, если она уже выходила или слишком близка по интенту к уже опубликованной.
- Каждая статья должна быть частью кластера, а не одиночным выстрелом в темноту.
- Решения принимаются по данным: Яндекс.Метрика + Яндекс.Вебмастер + keys.so.
- Следующий обязательный шаг развития аналитики: подключить Google Search Console и Google Analytics 4.

## 2. Цель и горизонт

### Цель
Вывести `delay.site` на 1000+ уников в день.

### Горизонт
- 30 дней — укрепление структуры, кластеров, контентной машины и аналитики
- 90 дней — заметный рост поискового трафика и первых сильных кластеров
- 6–12 месяцев — достижение 1000+ уников/день при дисциплинированном исполнении

## 3. Источники данных

### Обязательные источники
1. **Яндекс.Метрика** — трафик, страницы входа, поведение, конверсионные сигналы
2. **Яндекс.Вебмастер** — показы, клики, индекс, запросы, проблемы, рост/падение
3. **keys.so** — кластеры, частотность, конкуренты, gap analysis, приоритеты тем

### Подключить дополнительно
4. **Google Search Console** — запросы, страницы, покрытие, CTR, позиции в Google
5. **Google Analytics 4** — дополнительный слой поведения и атрибуции

## 6. Канонический pipeline статьи

1. Выбор темы через Brave Search + keys.so
2. Обязательная сверка с `memory/delay-site-content-log.md`
3. Отсев дублей и близких интентов
4. Research-step через **Perplexity Sonar via OpenRouter**
5. Написание статьи моделью текста
6. Генерация обложки через **OpenRouter → Gemini 3 Pro Image**
7. Конвертация `PNG -> JPG quality 85`
8. Публикация через WP REST API
9. URL брать только из `link` в API-ответе WordPress
10. Запись результата в `memory/delay-site-content-log.md`

## 7. Жёсткие правила контента

### Обязательно
- Первый абзац = прямой ответ на главный запрос
- H2 по возможности вопросами
- FAQ 5–7 вопросов
- Таблицы и списки
- Перелинковка на близкие материалы
- CTA в конце
- Только лёгкие JPG-обложки
- Только реальные, практичные темы

### Запрещено
- Дубли тем
- Каннибализация интентов
- Публикации без сверки с логом
- AI-темы ради AI-тем
- Ручная сборка URL из slug

## 8. Еженедельный ритм работы

Каждую неделю Кими делает:
1. Срез по Яндекс.Метрике
2. Срез по Яндекс.Вебмастеру
3. Срез по keys.so
4. Список страниц/запросов на рост
5. Выбор 1–3 старых статей для обновления
6. Корректировку тематических приоритетов

## 9. Что смотреть в метрике и вебмастере

### Метрика
- визиты
- пользователи
- просмотры
- отказы
- среднее время
- страницы входа
- источники трафика
- вклад новых статей

### Вебмастер
- показы
- клики
- CTR
- позиции
- индексируемые страницы
- исключённые страницы
- проблемы сайта
- запросы с показами, но слабым CTR

## 10. Что делать через keys.so

- собирать тематические кластеры
- находить НЧ/СЧ запросы
- искать конкурентов по кластерам
- выявлять контентные пробелы
- выбирать темы на 1–2 недели вперёд
- искать запросы для обновления старых статей

## 12. Операционное правило

Кими действует по этому плану ежедневно, корректируя темы и приоритеты по данным Метрики, Вебмастера и keys.so.

Если данные говорят, что стратегия буксует, корректируется не цель, а тактика.
